# Alpine Legacy — Foundation Architecture 0.6.1

## Цель

Ядро должно принимать сотни регионов, гор, маршрутов, организаций и NPC без переписывания карьеры, симуляции и интерфейса.

## Авторитетные реестры

`WorldEcosystem` хранит сущности в нормализованных таблицах:

- `regions.byId`;
- `mountains.byId`;
- `routes.byId`;
- `organizations.byId`;
- `npcs.byId`;
- `gear.byId`;
- `expeditionOffers.byId`.

Связи между сущностями хранятся через ID. Массив `region.mountains` временно сохранён как совместимая проекция для старых экранов и не является новым источником истины.

## Контент и состояние

`ecosystem.content` содержит определения: географию, биографии, параметры маршрутов и организаций.

`ecosystem.runtime` содержит изменяемое состояние: травмы, отношения, доступность, результаты гор, финансы организаций и открытые экспедиции.

Определение NPC не содержит его текущего здоровья. Runtime-запись NPC не дублирует имя и биографию.

## Карьера

Карьера хранит:

- членство или независимый статус;
- ранг и очки ранга;
- полномочия;
- выбранную заявку на экспедицию;
- известные игроку ID людей;
- собственное состояние и историю.

Новичок организации не получает личную команду. Он выбирает место в NPC-экспедиции. Независимый альпинист может идти один или присоединяться к чужому походу. Право выбирать команду и отдавать общие приказы открывается рангом.

## Маршруты

Каждый `ExpeditionRoute` имеет граф фаз, а не только линейный список участков:

- подход;
- базовый лагерь;
- переносы и акклиматизация;
- технические узлы;
- вершина;
- отдельный спуск;
- выход из района.

Текущий исполнитель восхождения пока использует совместимую линейную проекцию. Следующий модуль должен исполнять сам граф и наполнять его индивидуальными действиями, приказами и событиями.

## Валидация

`validateWorldEcosystem` проверяет:

- дубли ID;
- потерянные ссылки;
- маршрут без горы;
- NPC без организации;
- недостижимые узлы графа;
- отсутствующий спуск и выход;
- заявки с несуществующими участниками;
- слишком короткий бюджет игрового времени.

Генератор вызывает валидатор до возвращения мира.

## Добавление контента

Новый контент должен проходить один путь:

1. создать определения с устойчивыми ID;
2. собрать `EntityTable`;
3. связать сущности только через ID;
4. создать минимальное runtime-состояние;
5. запустить валидатор;
6. использовать селекторы вместо обхода вложенных массивов.

Ручной и процедурный контент должны выдавать одинаковые типы сущностей.
